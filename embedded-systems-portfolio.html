Arduino C++ task1_led_blink.ino 
// Task 1 – Basic Output Control
// LED Indicator System using Arduino UNO
// LED connected to Pin 13 through 220Ω resistor

const int LED_PIN = 13;  // Digital pin for LED

void setup() {
  pinMode(LED_PIN, OUTPUT);  // Set LED pin as output
  Serial.begin(9600);
  Serial.println("LED Indicator System Ready");
}

void loop() {
  // Turn LED ON
  digitalWrite(LED_PIN, HIGH);
  Serial.println("LED: ON");
  delay(1000);  // Wait 1 second

  // Turn LED OFF
  digitalWrite(LED_PIN, LOW);
  Serial.println("LED: OFF");
  delay(1000);  // Wait 1 second
}
schema

Circuit Explanation

1
Power Connection

Arduino 5V rail provides operating power. The GND pin serves as the common ground reference for the circuit.

2
Current-Limiting Resistor

A 220 Ω resistor is placed in series with the LED to limit current to ~18 mA, protecting both the LED and the Arduino output pin.

3
LED Wiring

The LED anode (+) connects through the resistor to Pin 13. The cathode (−) connects to GND. When Pin 13 is HIGH (5 V), current flows and the LED lights up.

4
Software Control

digitalWrite(LED_PIN, HIGH/LOW) toggles the output pin, turning the LED on and off with a 1-second interval.

// Task 2 – Variable Control System
// LED Brightness Control using PWM + Potentiometer
// Potentiometer on A0, LED on PWM Pin 9

const int POT_PIN = A0;   // Potentiometer analog input
const int LED_PIN = 9;    // PWM-capable pin

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);
  Serial.println("PWM Brightness Control Ready");
}

void loop() {
  int potValue = analogRead(POT_PIN);    // Read 0–1023

  // Map potentiometer value to PWM range (0–255)
  int brightness = map(potValue, 0, 1023, 0, 255);

  analogWrite(LED_PIN, brightness);  // Set LED brightness

  Serial.print("Pot: ");
  Serial.print(potValue);
  Serial.print(" | Brightness: ");
  Serial.println(brightness);

  delay(50);  // Small delay for stability
}
schema

Circuit Explanation

1
Potentiometer as Voltage Divider

The potentiometer's two outer pins connect to 5 V and GND. The wiper (middle pin) outputs a variable voltage between 0–5 V, which is read by Arduino's ADC on pin A0 as a value 0–1023.

2
PWM Output on Pin 9

Pin 9 is PWM-capable (~). analogWrite() generates a 490 Hz PWM signal whose duty cycle (0–255) controls the average LED voltage and thus perceived brightness.

3
Value Mapping

The map() function scales the 10-bit ADC range (0–1023) to the 8-bit PWM range (0–255) for proportional brightness control.

Arduino C++ task3_button_led.ino 
// Task 3 – User Interaction System
// Button Controlled LED Toggle with debounce
// Button on Pin 2, LED on Pin 13

const int BTN_PIN = 2;   // Push button input pin
const int LED_PIN = 13;  // LED output pin

bool ledState     = false;  // Current LED state
bool lastBtnState = false;  // Previous button reading
unsigned long lastDebounce = 0;
const int DEBOUNCE_MS = 50;  // Debounce delay in ms

void setup() {
  pinMode(BTN_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  bool btnRead = digitalRead(BTN_PIN);

  // Detect rising edge (press) with debounce
  if (btnRead != lastBtnState) {
    lastDebounce = millis();
  }

  if ((millis() - lastDebounce) > DEBOUNCE_MS) {
    if (btnRead == HIGH && lastBtnState == LOW) {
      ledState = !ledState;  // Toggle LED
      digitalWrite(LED_PIN, ledState ? HIGH : LOW);
      Serial.println(ledState ? "LED ON" : "LED OFF");
    }
  }

  lastBtnState = btnRead;
}
schema

Circuit Explanation

1
Pull-Down Resistor Configuration

The 10 kΩ resistor connects Pin 2 to GND, ensuring a stable LOW reading when the button is not pressed. Without it, the pin would "float" and give noisy readings.

2
Button Wiring

One side of the button connects to 5 V, the other to Pin 2. When pressed, 5 V is applied directly to the pin, overriding the pull-down to give a HIGH reading.

3
Debouncing

Mechanical buttons bounce 10–50 ms on contact. The software debounce waits 50 ms after any state change before registering a valid press, preventing multiple false triggers.

Arduino C++ task4_buzzer_alarm.ino 
// Task 4 – Alert / Notification System
// Buzzer Alarm System – button triggers buzzer alarm
// Button on Pin 2, Buzzer on Pin 8

const int BTN_PIN   = 2;   // Trigger button
const int BUZZER_PIN = 8;   // Buzzer output pin

void setup() {
  pinMode(BTN_PIN, INPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  Serial.begin(9600);
  Serial.println("Buzzer Alarm System Ready");
}

void loop() {
  int trigger = digitalRead(BTN_PIN);

  if (trigger == HIGH) {
    // Alarm pattern: three short beeps
    for (int i = 0; i < 3; i++) {
      tone(BUZZER_PIN, 1000);   // 1 kHz tone
      delay(200);
      noTone(BUZZER_PIN);
      delay(100);
    }
    Serial.println("ALARM TRIGGERED!");
  } else {
    noTone(BUZZER_PIN);  // No tone when idle
  }

  delay(50);
}
schema

Circuit Explanation

1
Active vs Passive Buzzer

An active buzzer produces sound when simply powered. A passive buzzer requires a frequency-modulated signal — use tone() for passive buzzers to specify the frequency.

2
Buzzer on Pin 8

Connect the buzzer's positive terminal to Pin 8 and the negative to GND. The tone(pin, frequency) function generates a square wave of the given frequency.

3
Alarm Pattern

The code creates a 3-beep alarm pattern using a loop — each beep is 200 ms on, 100 ms off. This pattern is easily recognisable as an alert signal.

// Task 5 – Display / Information System
// 16x2 LCD showing status and a counter
// Uses LiquidCrystal library (parallel 4-bit mode)
// RS=12, EN=11, D4=5, D5=4, D6=3, D7=2

#include <LiquidCrystal.h>

// Initialise LCD: RS, EN, D4, D5, D6, D7
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);

int counter = 0;

void setup() {
  lcd.begin(16, 2);         // 16 columns, 2 rows
  lcd.print("System Ready"); // Display welcome message
  delay(2000);
  lcd.clear();
}

void loop() {
  // Row 0: Status label
  lcd.setCursor(0, 0);
  lcd.print("Status: ACTIVE  ");

  // Row 1: Uptime counter
  lcd.setCursor(0, 1);
  lcd.print("Count: ");
  lcd.print(counter);
  lcd.print("        ");  // Clear old digits

  counter++;
  delay(1000);  // Update every second
}
schema

Circuit Explanation

1
Power & Backlight

LCD VSS → GND, VDD → 5 V, A (anode) → 5 V through 220 Ω, K (cathode) → GND. This powers the module and activates the backlight.

2
Contrast (V0)

A 10 kΩ potentiometer wiper connects to V0. Adjusting it sets the pixel contrast. In Wokwi this is typically set automatically.

3
4-Bit Parallel Mode

RS, EN, and data lines D4–D7 connect to Arduino pins. The library handles the 4-bit communication protocol, sending data in two nibbles per character.

Arduino C++ task6_ldr_sensor.ino 
// Task 6 – Environmental Sensing System
// LDR Light Detection – auto LED control based on light level
// LDR voltage divider on A0, LED on Pin 13
// In darkness: resistance HIGH → voltage LOW → LED ON
// In light:    resistance LOW  → voltage HIGH → LED OFF

const int LDR_PIN       = A0;   // LDR analog input
const int LED_PIN       = 13;   // LED output pin
const int DARK_THRESHOLD = 400;  // Values below = dark

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);
  Serial.println("LDR Environmental Sensing Ready");
}

void loop() {
  int lightLevel = analogRead(LDR_PIN);

  Serial.print("Light Level: ");
  Serial.print(lightLevel);

  if (lightLevel < DARK_THRESHOLD) {
    digitalWrite(LED_PIN, HIGH);  // Dark → LED ON
    Serial.println(" → DARK  | LED: ON");
  } else {
    digitalWrite(LED_PIN, LOW);   // Bright → LED OFF
    Serial.println(" → LIGHT | LED: OFF");
  }

  delay(200);  // Sample every 200 ms
}
schema

Circuit Explanation

1
LDR Voltage Divider

The LDR (top) and 10 kΩ fixed resistor (bottom) form a voltage divider between 5 V and GND. The midpoint voltage is read by A0. As light decreases, LDR resistance increases and midpoint voltage drops.

2
Threshold Comparison

The ADC value is compared against a DARK_THRESHOLD (400 out of 1023). Values below 400 indicate darkness; above indicate sufficient ambient light.

3
Automatic LED Response

The Arduino automatically turns the LED on when dark and off when bright — simulating how smart street lights and automatic indoor lighting systems operate.
